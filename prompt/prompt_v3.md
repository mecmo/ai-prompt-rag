# Docker 系统化学习（企业级讲师视角）

## ① Docker 定义

Docker 是一种**轻量级容器化平台**，用于将应用及其运行环境打包为“镜像（Image）”，并以“容器（Container）”形式运行。核心目标是解决“环境不一致”问题，实现应用在开发、测试、生产环境的**一致运行**，是云计算与 DevOps 的基础设施级技术。

---

## ② 为什么需要 Docker

在企业中，开发、测试、运维环境经常不一致，导致“能开发不能上线”。传统虚拟机资源重、启动慢，而 Docker 通过容器共享宿主机内核，实现秒级启动和高密度部署。同时支持标准化交付，使应用从“人工部署”变成“镜像交付”，大幅提升交付效率与稳定性。

---

## ③ 核心组件

Docker 体系由六部分组成：

* **Image（镜像）**：应用运行模板
* **Container（容器）**：镜像运行实例
* **Docker Engine**：核心运行与管理引擎
* **Dockerfile**：镜像构建脚本
* **Docker Hub**：公共镜像仓库
* **Docker Compose**：多容器编排工具
  它们共同构成“构建→运行→分发”完整链路。

---

## ④ 工作流程

典型流程为：
开发编写应用 → 编写 Dockerfile → 构建镜像（docker build） → 推送仓库（Docker Hub/私有仓库） → 服务器拉取镜像 → docker run 启动容器 → Compose/K8s 进行多服务编排。
企业中通常结合 CI/CD，实现代码提交后自动构建并部署容器。

---

## ⑤ 企业实际应用

Docker 广泛用于：

* Web 服务部署（Nginx、Flask、SpringBoot）
* 数据库快速部署（MySQL、Redis）
* 微服务架构（服务拆分独立容器）
* CI/CD 自动化流水线
* 云平台基础运行环境
  在云计算中，Docker 常作为 Kubernetes 的底层运行单元。

---

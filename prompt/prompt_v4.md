# Kubernetes

## ① 定义

**Kubernetes（简称 K8s）** 是一个开源的容器编排平台，用于自动管理 Docker 等容器化应用。它能够实现容器的自动部署、扩缩容、故障恢复、负载均衡和滚动更新，帮助企业高效管理大量容器，是云原生架构的核心技术，也是现代云计算运维的重要基础。

---

## ② 为什么需要 Kubernetes

当应用数量增加到几十甚至上百个容器时，仅依靠 Docker 已难以完成统一管理。Kubernetes 能自动调度容器运行、监控服务状态、故障自动恢复，并支持弹性伸缩和滚动升级，大幅提升系统的可用性、稳定性和运维效率，因此成为企业容器集群管理的主流方案。

---

## ③ 核心组件

Kubernetes 的核心组件包括：**Pod**（最小部署单元）、**Node**（工作节点）、**Cluster**（集群）、**Deployment**（应用部署与副本管理）、**Service**（服务发现与负载均衡）、**Ingress**（外部访问入口）以及 **Control Plane**（控制平面），共同完成容器集群管理。

---

## ④ 企业应用

Kubernetes 广泛应用于微服务架构、云平台、大模型平台和 DevOps 场景。企业通常使用 K8s 管理 Web 服务、数据库、中间件及 AI 推理服务，实现自动部署、弹性扩容、滚动更新和高可用运行，也是各大公有云 Kubernetes 服务的基础。

---

## ⑤ 面试重点

重点掌握 Kubernetes 与 Docker 的关系、Pod 和 Container 的区别、Deployment 与 Service 的作用、Pod 调度流程、Service 服务发现、Ingress 路由、自动扩缩容（HPA）以及滚动更新机制。同时了解 Kubernetes 在云原生和 AI 平台运维中的典型应用。

---

## 学习路线

**容器基础回顾 → Kubernetes 架构 → Pod → Deployment → Service → Ingress → ConfigMap 与 Secret → Volume 存储 → Helm → HPA 自动扩缩容 → Kubernetes 项目部署 → 云平台 Kubernetes（如华为云 CCE） → 面试专项训练**

## 输出效果分析

### 本次优化内容

本轮实验增加了 Few-shot（示例引导）。

通过提供一份完整的 Docker 示例，引导 AI 学习输出格式，而不是仅依赖文字描述。

相比 V3，本轮回答格式更加统一。

- 章节顺序

### 本轮结论

Few-shot Prompt 可以通过提供示例，让 AI 学习目标输出格式，从而显著提高回答的一致性和稳定性。

相比仅依靠文字描述，示例引导更加直观，更适合课程开发、知识库整理和批量内容生成等实际业务场景。
